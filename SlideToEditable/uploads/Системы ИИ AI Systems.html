1.1. Понятие и история ИИ

Система ИИ — программная система, имитирующая процесс мышления человека.

Ключевые вехи:

  • XIII в. Раймонд Луллий (механическое устройство)
  • 1940-е Норберт Винер (Кибернетика)
  • 1956 Термин «ИИ» (Дартмутский колледж, США)
  • 1970-е Экспертные системы (Mycin, Dendral)

1.1. Concept and History of AI

AI System — a software system that simulates human thinking processes.

Key milestones:

  • 13th century Ramon Llull (mechanical device)
  • 1940s Norbert Wiener (Cybernetics)
  • 1956 Term "AI" coined (Dartmouth College, USA)
  • 1970s Expert systems (Mycin, Dendral)

1.1 人工智能的概念与历史

人工智能系统 — 模拟人类思维过程的软件系统。

关键里程碑：

  • 13世纪 雷蒙·卢利（机械装置）
  • 1940年代 诺伯特·维纳（控制论）
  • 1956年 "人工智能"术语提出（美国达特茅斯学院）
  • 1970年代 专家系统（Mycin, Dendral）

Два направления ИИ

Нейрокибернетика

«Единственный объект, способный мыслить — мозг».

  • Моделирование структуры мозга
  • Нейроны и нейронные сети
  • Аппаратная реализация (нейрочипы)

Кибернетика «чёрного ящика»

«Не важно устройство, важен результат».

  • Поиск алгоритмов решения задач
  • Логический вывод, эвристики
  • Языки: Prolog, Lisp

Two Directions of AI

Neurocybernetics

"The only object capable of thinking is the brain."

  • Modeling brain structure
  • Neurons and neural networks
  • Hardware implementation (neurochips)

"Black Box" Cybernetics

"The device doesn't matter, the result does."

  • Searching for problem-solving algorithms
  • Logical inference, heuristics
  • Languages: Prolog, Lisp

人工智能的两个方向

神经控制论

"唯一能够思考的对象是大脑。"

  • 模拟大脑结构
  • 神经元与神经网络
  • 硬件实现（神经芯片）

"黑箱"控制论

"装置不重要，结果才重要。"

  • 寻找问题求解算法
  • 逻辑推理，启发式方法
  • 语言：Prolog, Lisp

  • 1954 Семинар «Автоматы и мышление» в МГУ (А. А. Ляпунов)
  • Программа АЛПЕВ ЛОМИ: автоматическое доказательство теорем
  • Алгоритм «Кора»: М. М. Бонгард (распознавание образов)
  • 1974 Научный совет по проблеме «ИИ» (Д. А. Поспелов)
  • 1988 Ассоциация искусственного интеллекта (АИИ)

1.1.2. Artificial Intelligence in Russia

  • 1954 Seminar "Automata and Thinking" at MSU (A. A. Lyapunov)
  • ALPEV LOMI program: automated theorem proving
  • "Kora" algorithm: M. M. Bongard (pattern recognition)
  • 1974 Scientific Council on "AI" (D. A. Pospelov)
  • 1988 Association for Artificial Intelligence (AAI)

1.1.2 俄罗斯的人工智能

  • 1954年 莫斯科大学"自动机与思维"研讨会（А. А. Ляпунов）
  • АЛПЕВ ЛОМИ程序：自动定理证明
  • "Кора"算法：М. М. Бонгард（模式识别）
  • 1974年 "人工智能"问题科学委员会（Д. А. Поспелов）
  • 1988年 人工智能协会（АИИ）

1.1.3. Функциональная структура системы ИИ

Система состоит из трёх комплексов:

  1. Исполнительная система (ИС): выполняет программы, решает задачи
  2. Интеллектуальный интерфейс: адаптация под пользователя, общение на естественном языке
  3. База знаний (БЗ): центральное звено. Хранит знания о проблемной среде

1.1.3. Functional Structure of AI System

The system consists of three components:

  1. Executive System (ES): executes programs, solves problems
  2. Intelligent Interface: user adaptation, natural language communication
  3. Knowledge Base (KB): central component. Stores knowledge about the problem domain

1.1.3 人工智能系统的功能结构

系统由三个组件构成：

  1. 执行系统（ES）： 执行程序，解决问题
  2. 智能接口： 用户适配，自然语言交互
  3. 知识库（KB）： 核心组件。存储问题领域的知识

1.2. Направления развития ИИ

  • Представление знаний: ядро экспертных систем
  • Игровые задачи: шахматы, шашки (лабиринтная модель + эвристики)
  • Естественно-языковые интерфейсы: машинный перевод, анализ текста
  • Распознавание образов: близко к машинному обучению
  • Интеллектуальные роботы: от жёстких схем до самоорганизующихся систем
  • Обучение и самообучение: накопление знаний на основе данных

1.2. Directions of AI Development

  • Knowledge Representation: core of expert systems
  • Game Tasks: chess, checkers (maze model + heuristics)
  • Natural Language Interfaces: machine translation, text analysis
  • Pattern Recognition: close to machine learning
  • Intelligent Robots: from rigid schemes to self-organizing systems
  • Learning & Self-learning: knowledge accumulation based on data

1.2 人工智能的发展方向

  • 知识表示： 专家系统的核心
  • 博弈任务： 国际象棋、跳棋（迷宫模型+启发式）
  • 自然语言接口： 机器翻译、文本分析
  • 模式识别： 接近机器学习
  • 智能机器人： 从刚性方案到自组织系统
  • 学习与自学习： 基于数据的知识积累

1.3. Данные и Знания

Данные

Отдельные факты, характеризующие объекты и процессы.

Хранятся в Базах Данных (БД).

Знания

Выявленные закономерности (принципы, связи, законы).

Результат мыслительной деятельности.

Хранятся в Базах Знаний (БЗ).

Главное отличие: Из БД можно извлечь только заложенные факты. Из БЗ можно выводить новые факты, используя закономерности.

1.3. Data and Knowledge

Data

Individual facts characterizing objects and processes.

Stored in Databases (DB).

Knowledge

Identified patterns (principles, connections, laws).

Result of cognitive activity.

Stored in Knowledge Bases (KB).

Key Difference: From DB you can only retrieve stored facts. From KB you can infer new facts using patterns.

1.3 数据与知识

数据

描述对象和过程的个别事实。

存储在数据库（DB）中。

知识

发现的规律（原则、联系、法则）。

思维活动的结果。

存储在知识库（KB）中。

主要区别： 从数据库只能检索已存储的事实。从知识库可以推导新事实，利用规律。

1.3.2. Модели представления знаний

1. Семантические сети

Ориентированный граф. Вершины — понятия, дуги — отношения («это», «имеет частью»).

2. Фреймы (М. Минский)

Единица представления знания («рамка» ситуации).

  • Состоит из слотов (атрибутов)
  • Поддерживает наследование свойств (АКО-связи)
  • Уровни: скелетный, прототип, экземпляр

1.3.2. Knowledge Representation Models

1. Semantic Networks

Directed graph. Nodes — concepts, edges — relations ("is-a", "has-part").

2. Frames (M. Minsky)

Unit of knowledge representation ("frame" of a situation).

  • Consists of slots (attributes)
  • Supports inheritance of properties (AKO-links)
  • Levels: skeletal, prototype, instance

1.3.2 知识表示模型

1. 语义网络

有向图。节点—概念，边—关系（"是一种"，"有部分"）。

2. 框架（M. Minsky）

知识表示单元（情境的"框架"）。

  • 由槽（属性）组成
  • 支持属性的继承（AKO-链接）
  • 层级：骨架、原型、实例

Модели представления знаний (продолжение)

3. Формальные логические модели

Основаны на аксиомах и правилах вывода. Формальная система M=<T, P, A, B>.

4. Продукционные модели

Правила вида: «Если (условие), то (действие)».

  • База знаний = набор правил
  • Управляет перебором Машина вывода
  • Вывод: прямой или обратный

Knowledge Representation Models (cont.)

3. Formal Logical Models

Based on axioms and inference rules. Formal system M=<T, P, A, B>.

4. Production Models

Rules of the form: "If (condition), then (action)."

  • Knowledge Base = set of rules
  • Controlled by Inference Engine
  • Inference: forward or backward chaining

知识表示模型（续）

3. 形式逻辑模型

基于公理和推理规则。形式系统 M=<T, P, A, B>。

4. 产生式模型

规则形式："如果（条件），则（动作）"。

  • 知识库 = 规则集
  • 由推理机控制遍历
  • 推理：正向或反向链

1.4. Экспертные системы (ЭС)

ЭС — сложные программные комплексы, аккумулирующие знания специалистов и тиражирующие опыт для консультаций.

Отличия от систем обработки данных:

  • Символьный способ представления (не числовой)
  • Символьный вывод
  • Эвристический поиск решения
  • Способность пополнять знания

1.4. Expert Systems (ES)

ES — complex software systems that accumulate expert knowledge and replicate experience for consultations.

Differences from data processing systems:

  • Symbolic representation (not numeric)
  • Symbolic inference
  • Heuristic problem solving
  • Ability to expand knowledge

1.4 专家系统（ES）

专家系统 — 积累专家知识并复制经验用于咨询的复杂软件系统。

与数据处理系统的区别：

  • 符号表示（非数值）
  • 符号推理
  • 启发式问题求解
  • 知识扩展能力

Структура экспертной системы

Типичная статическая ЭС состоит из:

  1. Решатель: формирует последовательность правил
  2. Рабочая память (БД): хранит текущие данные задачи
  3. База знаний (БЗ): хранит долгосрочные знания и правила
  4. Компонент приобретения знаний: наполнение системы экспертом
  5. Объяснительный компонент: поясняет, как получено решение
  6. Диалоговый компонент: интерфейс с пользователем

Expert System Structure

A typical static ES consists of:

  1. Inference Engine: forms sequence of rules
  2. Working Memory (DB): stores current task data
  3. Knowledge Base (KB): stores long-term knowledge and rules
  4. Knowledge Acquisition: system population by expert
  5. Explanation Component: explains how solution was obtained
  6. Dialogue Component: user interface

专家系统结构

典型静态专家系统由以下部分组成：

  1. 推理机： 形成规则序列
  2. 工作内存（DB）： 存储当前任务数据
  3. 知识库（KB）： 存储长期知识和规则
  4. 知识获取组件： 专家填充系统
  5. 解释组件： 解释如何获得解决方案
  6. 对话组件： 用户界面

Разработка и классификация ЭС

Участники: Эксперт, Инженер по знаниям, Программист

Режимы работы: Приобретение знаний, Консультация

Классификация по задаче:

  • Интерпретация, Диагностика, Мониторинг
  • Проектирование, Прогнозирование, Планирование, Обучение

ES Development & Classification

Participants: Expert, Knowledge Engineer, Programmer

Operating Modes: Knowledge Acquisition, Consultation

Classification by Task:

  • Interpretation, Diagnosis, Monitoring
  • Design, Prediction, Planning, Training

专家系统开发与分类

参与者： 专家、知识工程师、程序员

工作模式： 知识获取、咨询

按任务分类：

  • 解释、诊断、监控
  • 设计、预测、规划、培训

Инструментальные средства построения ЭС

  • Традиционные языки: C, C++, Pascal (высокая эффективность, но сложность)
  • Языки ИИ: Lisp, Prolog (богатые возможности для символьных данных)
  • Спец. инструментарий: Библиотеки над Lisp (KEE, ARTS)
  • «Оболочки» (Shells): «Пустые» ЭС без базы знаний (например, EMYCIN)

Tools for Building Expert Systems

  • Traditional Languages: C, C++, Pascal (high efficiency, but complex)
  • AI Languages: Lisp, Prolog (rich capabilities for symbolic data)
  • Special Toolkits: Libraries over Lisp (KEE, ARTS)
  • "Shells": "Empty" ES without knowledge base (e.g., EMYCIN)

构建专家系统的工具

  • 传统语言： C, C++, Pascal（高效但复杂）
  • AI语言： Lisp, Prolog（丰富的符号数据处理能力）
  • 专用工具包： Lisp库（KEE, ARTS）
  • "外壳"： 无知识库的"空"专家系统（如EMYCIN）

Технология разработки ЭС (6 этапов)

  1. Идентификация: осмысление задач, формирование требований
  2. Концептуализация: анализ предметной области, выявление понятий
  3. Формализация: выбор способа представления знаний
  4. Выполнение: создание прототипа ЭС, наполнение БЗ
  5. Тестирование: оценка работы системы на примерах
  6. Опытная эксплуатация: проверка пригодности для пользователя

ES Development Technology (6 Stages)

  1. Identification: task understanding, requirements formation
  2. Conceptualization: domain analysis, concept identification
  3. Formalization: selecting knowledge representation method
  4. Implementation: ES prototype creation, KB population
  5. Testing: system evaluation on examples
  6. Experimental Operation: user suitability verification

专家系统开发技术（6个阶段）

  1. 识别： 任务理解，需求形成
  2. 概念化： 领域分析，概念识别
  3. 形式化： 选择知识表示方法
  4. 实现： 专家系统原型创建，知识库填充
  5. 测试： 基于示例的系统评估
  6. 试运行： 用户适用性验证
