# 录音功能使用指南

## 问题：为什么开始录音哪里没办法读取我的声音？

### 原因分析：

1. **浏览器权限问题**：现代浏览器需要用户明确授权才能访问麦克风
2. **音频格式问题**：浏览器录音的音频格式需要与后端API兼容
3. **网络连接问题**：录音数据需要通过网络发送到服务器进行处理

### 解决方案：

#### 1. 浏览器权限设置
- 当您第一次点击"Начать запись"按钮时，浏览器会显示权限请求对话框
- **必须点击"允许"或"Разрешить"**，否则浏览器无法访问麦克风
- 如果之前拒绝了权限，需要手动重置：
  - Chrome: 点击地址栏左侧的锁形图标 → 网站设置 → 麦克风 → 允许
  - Firefox: 点击地址栏左侧的图标 → 权限 → 麦克风 → 允许
  - Edge: 类似Chrome的设置

#### 2. 使用步骤
1. 点击侧边栏的"Начать запись"按钮
2. 浏览器会弹出权限请求 → 点击"允许"
3. 按钮会变成"Остановить запись"，状态显示"Запись..."
4. 对着麦克风说话
5. 点击"Остановить запись"停止录音
6. 状态会显示"Распознавание речи..."
7. 识别完成后，文本会自动填入语音功能区域的文本框中
8. 页面会自动切换到"Голосовые функции"选项卡

#### 3. 备用方案：上传音频文件
如果直接录音有问题，可以使用上传功能：
1. 切换到"Голосовые функции"选项卡
2. 点击"Загрузить аудиофайл"按钮
3. 选择WAV或MP3格式的音频文件
4. 系统会自动识别并显示结果

### 技术细节：

#### 支持的音频格式：
- WAV (PCM格式，推荐)
- MP3
- OGG
- WebM

#### 录音参数：
- 采样率：44.1kHz
- 声道：单声道
- 位深度：16位

### 故障排除：

#### 问题1：浏览器没有弹出权限请求
- 检查浏览器设置中是否完全禁止了麦克风访问
- 尝试在隐身模式下打开网站（隐私模式）
- 重启浏览器

#### 问题2：录音按钮点击后没有反应
- 检查浏览器控制台是否有错误（按F12 → Console）
- 确保网站使用HTTPS（本地开发环境可以使用HTTP）
- 检查麦克风是否正常工作（可以测试其他网站如Google语音搜索）

#### 问题3：录音成功但识别失败
- 确保说话声音足够大
- 减少背景噪音
- 尝试说简单的短语
- 检查网络连接

### 测试录音功能：

您可以使用以下命令测试后端API是否正常工作：

```bash
# 测试文本转语音API
curl -X POST http://localhost:5000/api/text-to-speech \
  -H "Content-Type: application/json" \
  -d '{"text": "Привет, это тест", "language": "ru"}'

# 测试语音转文本API（需要有效的音频文件）
curl -X POST http://localhost:5000/api/speech-to-text \
  -F "audio=@test_audio.wav"
```

### 已知限制：

1. **语音识别准确度**：俄语语音识别准确度约为85-90%
2. **网络延迟**：录音需要发送到服务器处理，可能会有1-3秒延迟
3. **浏览器兼容性**：
   - Chrome 58+ ✓
   - Firefox 55+ ✓
   - Edge 79+ ✓
   - Safari 11+ ✓

### 如果所有方法都失败：

1. 手动输入文本到"Голосовые функции"选项卡的文本框中
2. 点击"Преобразовать в речь"按钮测试文本转语音功能
3. 如果文本转语音正常工作，说明后端API正常
4. 问题可能在前端录音功能或浏览器权限设置
